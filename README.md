# EduPlatform

**Интерактивна уеб платформа за дигитално обучение, управление на уроци, тестове и академичен напредък**

**Важно: В архива се намира целият проект, т.е. не е нужно да се клонира GitHub хранилище, за да бъде стартиран локално.**

---

## Общ преглед на проекта

Съвременното обучение все по-често изисква достъпна, структурирана и надеждна дигитална среда, в която ученици, учители и администратори да могат да работят спокойно и ефективно. В практиката често възниква проблем с разпокъсано учебно съдържание, липса на централизирани тестове и затруднено проследяване на резултатите и прогреса.

В отговор на тази нужда проектът **EduPlatform** има за цел разработването на цялостна образователна уеб платформа, която обединява управление на учебни предмети, уроци, тестове, резултати и потребители в една модерна система. Платформата предоставя отделни работни пространства за ученици, учители и администратори, като съчетава удобен интерфейс, статистики, многоезична поддръжка и възможност за бъдещо разширяване.

---

## Основни функционалности

### Потребители и роли
- Регистрация, вход и удостоверяване с JWT
- Отделни роли за **Администратор**, **Учител** и **Ученик**
- Потребителски профили и ролево-базиран достъп
- Одобрение на учителски акаунти от администратор

### Учебно съдържание
- Управление на учебни предмети по клас и паралелка
- Създаване, редактиране и изтриване на уроци
- Поддръжка на текст, изображения, прикачени файлове и YouTube линкове
- Филтриране на съдържание по предмет, клас и секция

### Тестове и оценяване
- Създаване на тестове към конкретни уроци
- Поддръжка на различни типове въпроси
- Автоматично оценяване на отговорите
- Запазване на резултатите и преглед на детайлна обратна връзка

### Статистики и проследяване
- Административно табло със статистики за платформата
- Учителски dashboard с преглед на предмети, уроци, тестове и ангажираност
- Ученически преглед на резултати и напредък
- Live activity feed и визуализации с графики

### Допълнителни възможности
- Многоезична поддръжка на български и английски
- Персонализация на интерфейса
- Glassmorphism UI и модерна responsive визия
- Качване и обслужване на файлове

## Структура на проекта

Проектът е разделен на две основни части:

- `EduPlatform.API` – ASP.NET Core Web API backend
- `eduplatform-web` – React + TypeScript frontend

Допълнително:

- `EduPlatform.API.Tests` – автоматизирани backend тестове
- `start-backend-clean.sh` – бърз старт на backend при проблем със stuck процеси
- `start-backend-rebuild.sh` – clean rebuild и нов старт на backend
- `normalize_lesson_test_class_data.sql` и `seed_*.sql` – SQL помощни/seed файлове

## Как да стартираме проекта

**Важно: За локално стартиране трябва да имате инсталирани .NET SDK 7.0+, Node.js 20+ и npm.**

### 1. Инсталация

Ако работите с архива на проекта, тази стъпка може да бъде пропусната.

Ако все пак използвате Git хранилище:

```bash
git clone <repository-url>
cd diplomenproekt
```

### 2. Локално стартиране

Трябва да стартирате backend и frontend в отделни терминали.

#### 2.1. Стартиране на backend

Преминете в backend директорията:

```bash
cd EduPlatform.API
```

Проверете дали проектът се build-ва:

```bash
dotnet build
```

За нормално стартиране:

```bash
dotnet run --launch-profile http
```

По подразбиране backend API работи на:

```text
http://localhost:5067
```

Swagger интерфейсът е достъпен на:

```text
http://localhost:5067/swagger
```

#### 2.2. Стартиране на frontend

В отделен терминал преминете в frontend директорията:

```bash
cd eduplatform-web
```

Инсталирайте зависимостите:

```bash
npm install
```

Проверете дали няма build грешки:

```bash
npm run build
```

Стартирайте development сървъра:

```bash
npm run dev
```

При успешно стартиране frontend приложението обикновено е достъпно на:

```text
http://localhost:5173
```

**Важно: Ако порт `5173` не е свободен, Vite ще стартира на друг свободен порт.**

## Стартиране на backend при проблем със stuck процеси

Ако `dotnet run` забива, стои прекалено дълго на `Building...` или има стар блокирал процес:

### Бърз старт без излишен rebuild

От основната папка на проекта:

```bash
./start-backend-clean.sh
```

Скриптът:
- спира стари `EduPlatform.API` / `dotnet run` процеси
- използва съществуващ build, ако има такъв
- стартира backend-а отново
- ако порт `5067` е блокиран, преминава към `5080`

### Пълен clean rebuild

Ако искате напълно чисто преизграждане:

```bash
./start-backend-rebuild.sh
```

Този скрипт:
- спира backend процесите
- изчиства `bin/` и `obj/`
- build-ва проекта отначало
- стартира API-то отново

## Автоматични тестове

Backend тестовете се намират в:

```text
EduPlatform.API.Tests
```

За да ги стартирате:

```bash
dotnet test
```

Можете да изпълните командата от root директорията на проекта или от `EduPlatform.API.Tests`.

## База данни и SQL файлове

Проектът използва SQLite база данни. В backend директорията вече се съдържа работна база:

```text
EduPlatform.API/eduplatform.db
```

Освен това в root директорията има SQL файлове за помощни операции и seed на уроци, например:

- `normalize_lesson_test_class_data.sql`
- `seed_first_5_lessons.sql`
- `seed_lessons_all_44_grade8_a.sql`
- `seed_lessons_*.sql`

Тези файлове могат да се използват при нужда от допълнително зареждане или корекция на данни в базата.

## Препоръчителен ред за стартиране

За най-стабилен локален старт:

1. Стартирайте backend-а
```bash
cd EduPlatform.API
dotnet run --launch-profile http
```

2. Стартирайте frontend-а
```bash
cd eduplatform-web
npm install
npm run dev
```

3. Отворете приложението в браузър
```text
http://localhost:5173
```

Ако backend има проблеми със stuck процеси, използвайте:

```bash
./start-backend-clean.sh
```

## Кратко описание на ролите

- **Администратор** – управлява потребители, предмети, уроци, тестове и статистики
- **Учител** – работи с присвоени предмети, уроци и тестове
- **Ученик** – разглежда учебно съдържание, решава тестове и следи резултатите си
